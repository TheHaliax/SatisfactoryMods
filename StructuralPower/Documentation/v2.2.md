# v2.2.0 ‚Äî Lighting & Id config

Shipped **2026-07-05**. Opt-in category ‚Äî **Structural lighting** defaults **OFF** on new installs.

## Enable structural lighting

| Surface | Action |
|---------|--------|
| Pause ‚Üí Mods ‚Üí Structural Power ‚Üí **Debug** | Toggle **Structural lighting** |
| Chat (host) | `!lighting` |
| Console / cfg | `StructuralPower.Set GroupLighting 1` or `"GroupLighting": true` in `StructuralPower.cfg` |

Without Group Lighting enabled, lights behave vanilla (wire required). Poles, switches, and hoverpack from v2.1 are unchanged.

## Id config panel (I key)

Look at an eligible buildable and press **I**:

| Target | Source | Control |
|--------|--------|---------|
| **Light** | Where power comes from (structural default, switch id, or light group) | ‚Äî |
| **Power switch** | Structural feed | Switch gate id (building tag or override) |
| **Lights control panel** | Structural feed or switch id | Light group name (e.g. `Light1`) |

Ids are scoped to the **structure component** (connected island on the bus). Dropdowns list ids already in use on that island plus the component default.

Apply sends an RCO to the server; overrides persist in the **world save**.

## Named light groups (M4)

1. Enable Group Lighting.
2. Place lights control panel on powered structure.
3. Press **I** on the panel ‚Äî set **Control** to a group name (e.g. `Light1`).
4. Press **I** on each light ‚Äî set **Source** to the same name.
5. Vanilla **E** on the panel still adjusts only lights in that keyed group ‚Äî not every light on the structure.

## Switch subnets

When **Gate power switches** and **Mode B** (`PowerSwitchManualGroups: 1`) are on:

- Assign a switch **Control** id (e.g. `Switch1`).
- Set a panel or light **Source** to `Switch1`.
- Keyed consumers power only when that switch is **ON**.

## Architecture notes (for v2.3+)

v2.2 extends the v2.1 **Source/Control** model on the `Light` and `Switch` channels. The router also reserves `Generator`, `Extractor`, `Manufacturer`, `Transport`, and `Misc` channels for future opt-in features.

| Mechanism | Reuse in later versions |
|-----------|-------------------------|
| `FStructuralEndpointOverrides` (save) | Any buildable |
| `CollectIdsOnComponent` id pools | Add namespace buckets per feature |
| `ResolveSubnetFeedConnector` | Switch-gated sources for any device |
| `FStructuralDeviceAttach` | Consumer hidden-link pattern |
| Pre-`BeginPlay` mod component strip | Any transient outlet bus |

Lighting-specific: `PanelControlBus`, `FStructuralPanelControlledSync`, panel listener ‚Äî not reused by generators; same graph/RCO/hook patterns apply.
